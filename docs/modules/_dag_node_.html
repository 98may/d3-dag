Module "dag/node"

A Dag is simply a collection of DagNodes, defined by every reachable child node from the current returned node. If a DAG contains multiple roots, then the returned node will be a DagRoot that links to all nodes. Each child node on its own will function as a valid DAG with a single root. All DAGs are also iterators over all of their nodes.

Three methods exist to turn existing data into Dags:

  1. dagHierarchy - when the data already has a dag structure.
  2. dagStratify - when the dag has a tabular structure, referencing parents by id.
  3. dagConnect - when the dag has a link structure and is specified as pairs of nodes.

Methods names preceeded by an i will return a FluentIterable which is a wrapper around native EMCA iterators that also adds most methods found in the Array prototype making them much more useful for fluent functional programming.
